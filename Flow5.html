Portfolio Development - flow 5

This project was about taking our earlier portfolio and making it better, and documenting the process along the way.

What I did

The first thing I did was project planning, where I used trello to give me an idea of how I would like to use my time in the project duration. After that I made some information architecture and began to get inspired by other portfolios.

Then I made some design sketches, wireframes, and mockups as you can see on the right and below. I quickly got the idea of how I wanted the webpage to look. I wanted to make a simple design, that you could easily navigate and would keep you oriented at all time. Therefore, I went with natural colours and high contrast, the user would draw attention to the right things more easily. This also seem a little more professional in my opinion.

You can see how the sketch almost became the layout of the finished page. I only changed a bit on the project part. I wanted it to be a bit more visual pleasing than just images beside each other, so I used the idea behind my earlier portfolio. The wireframe and early mockups were almost as the final site came out, but their where some minor changes, as I got inspired along the way.

The last thing I did was to validate the code using "validator.w3.org", which after some small changes became error-free.
